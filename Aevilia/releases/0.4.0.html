Aevilia Release 0.4.0

Although it is still being developed, this version adds a TON of content ! Even more than 0.3.0.

Changelog symbol table

SymbolType

Changelog

*Heavy rewriting and smaller tweaks to the graphical engine, with the accompanying FPS gains ! Apparently this divided the rendering overload by 2 !
*The exterior plains and house have been extended horizontally to test camera scrolling.
*The door animation has been changed... multiple times, including fixing graphical oddities.
+A failsafe for invalid items has been added ; and because I love Pokémon glitches, it's called "MISSINGITEM.".
+It also has custom behaviour and description.
!Note that it doesn't exist, but is more of a collection of failsafes that prevent invalid behaviour from happening :)
+Adding a fighting engine.
+The second item of this game's : a Zap Toy !
*The "DESCRIBE" option has been replaced with a description appearing below the player's inventory.
+Instead is a nice "TOSS" option !
*Sprites and map layers aren't all loaded in memory at game startup, instead they are stored in external files and read as needed during map loading.
The external files are binary files generated from text files using a program ; the compilation is done by "make layers" and "make sprites" !
!As a result, disk space and RAM usage were reduced. Oh, and loading times are pretty much invisible - and I don't have a SSD.
~The player doesn't appear anymore on the crash screen.
+Alt+F4 / closing the window were changed to instantly shut the game down, but using Escape instead plays a small animation.
+Save points let you return to the title screen, even if you don't choose to save.
+Some command line arguments are supported by the game : -d enables debug info, and -g disables graphical acceleration.
+New debug info : colored bars display the overload (measured in CPU clock cycles) of different game components.
+Due to better knowledge of optimizations provided by GCC, code has been globally optimized.
~Obtaining an item blanked the description of the inventory's CANCEL button. This has been patched.
*The item debugging menu has been improved.
+Menus opening and closing now have an animation.
!I've noticed a bug sometimes occurs with the "options" command, where the cursor doesn't become white after an option has been selected. I don't know why this occurs.
